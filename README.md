# IoT Simulation as a temperature and humidity reader

This is a simple project where I simulate generating temperature and humidity data and sending it to ThingSpeak using an API.

I built this in the uni, while learning how APIs and distributed systems actually work in practice.

---

## ⚙️ What it does

* Generates random temperature and humidity values
* Sends data to ThingSpeak every 15 seconds
* Displays the data in real-time charts

---

## 🧠 What I learned

* How to work with APIs using Python
* How data is sent using query parameters (`field1`, `field2`)
* Basic concepts of distributed systems
* How IoT platforms like ThingSpeak receive and store data
* Why it's important to hide API keys using `.env`

---

## 🔐 One of the Moste Important Notes:

The API key is hidden using environment variables for security reasons!
